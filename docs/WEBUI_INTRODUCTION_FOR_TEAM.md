## このシステムについて

**CareerNavigator** は、スキルノートのデータを活用して、技術者の最適なキャリアパスを提案するWebアプリケーションです。

### 一言で言うと

> **「どのスキルを、どの順番で学べば、目標とするキャリアに効率的に到達できるか」を教えてくれるシステムです。**

### 特徴

- 🧠 **因果推論技術**を使った科学的なスキル推薦
- 📊 **個人向けキャリアダッシュボード**で自分の成長を可視化
- 🗺️ **組織スキルマップ**で組織全体のスキル状況を把握
- 📈 **スマートロードマップ**で学習計画を自動生成

## 取り組みの背景

### なぜこのシステムが必要だったのか

技術者のキャリア開発において、以下のような状況がありました：

#### 1. スキル習得の順序が不明確

```
❓ 「機械学習を学びたいけど、その前に何を学ぶべき?」
❓ 「この資格を取るには、どんな前提知識が必要?」
❓ 「目標とする先輩のスキルセットに近づくには?」
```

多くの技術者が、**何をどの順番で学べば効率的か**がわからず、試行錯誤していました。

#### 2. 組織全体のスキル状況が見えない

```
❓ 「チームにどんなスキルが不足している?」
❓ 「この技術を教えられる人は誰?」
❓ 「後継者として育成すべき人材は?」
```

マネージャー層も、組織全体のスキル分布を把握するのが困難でした。

#### 3. 個人のキャリアビジョンが描きにくい

```
❓ 「目指す役職に必要なスキルは何?」
❓ 「現在の自分とのギャップはどれくらい?」
❓ 「どのくらいの期間で目標に到達できる?」
```

具体的なキャリアパスが見えず、モチベーション維持が難しい状況でした。

## 解決したい課題

### 従来の推薦システムの問題点

従来のスキル推薦システムは、主に**「相関関係」**に基づいていました。

#### 相関関係ベースの推薦とは

```
例: 「Pythonを学んだ人は、データ分析も学んでいます」
   → だからあなたにもデータ分析を推薦します
```

**問題点:**
- ❌ なぜそのスキルを推薦するのか説明が不十分
- ❌ 学習の順序が考慮されていない
- ❌ 「Pythonの後にデータ分析」なのか「データ分析の後にPython」なのか不明
- ❌ 個人の現在のスキルレベルが考慮されない

### 我々が目指したこと

**因果関係**に基づく推薦システムで、以下を実現：

```
✅ 「Pythonを学ぶと、データ分析の習得が25%容易になります」
✅ だから、まずPythonから始めましょう
✅ あなたの現在のスキルから見て、学習準備は80%整っています
```

**メリット:**
- ✅ 推薦理由が明確で納得感がある
- ✅ 学習の順序が科学的に決まる
- ✅ 個人の状況に合わせた最適な推薦
- ✅ 「なぜ今これを学ぶべきか」が理解できる

## 今回実施したこと

### 1. 因果推論ベースの推薦システム

#### 従来の方法 vs 因果推論

| 観点 | 従来の推薦 | 因果推論ベースの推薦 |
|------|-----------|-------------------|
| **関係性** | 相関（一緒に起こる） | 因果（原因と結果） |
| **説明** | 「この2つはよく一緒に学ばれます」 | 「Aを学ぶとBの習得が容易になります」 |
| **順序** | 考慮なし | 明確な学習順序 |
| **納得感** | 低い | 高い |

#### 3軸スコアリングシステム

推薦するスキルを、3つの観点から評価します：

**① 準備完了度 (Readiness Score)**
```
意味: 今、このスキルを学ぶ準備がどれくらい整っているか
例: 「あなたは既にPythonとデータ構造を習得しているので、
     機械学習の準備は85%整っています」
```

**② 習得確率 (Probability Score)**
```
意味: 現在の保有スキルから見て、習得できる確率
例: 「現在のスキルセットから見て、
     この資格を取得できる確率は78%です」
```

**③ 有用性 (Utility Score)**
```
意味: 将来の目標達成にどれだけ役立つか
例: 「このスキルを習得すると、目標とする役職に必要な
     5つのスキルの習得が容易になります」
```

**総合スコア = 準備完了度^0.3 × 確率^0.3 × 有用性^0.4**

有用性を最も重視（0.4の重み）して、将来のキャリア目標達成に貢献するスキルを優先的に推薦します。

### 2. スマートキャリアロードマップ

#### ガントチャート形式の学習計画

依存関係を考慮した、視覚的にわかりやすい学習計画を自動生成します。

```
例:
┌─────────────────────────────────────────┐
│ Month 1-2:  Python基礎                   │
└─────────────────────────────────────────┘
                ↓
        ┌───────────────────┐
        │ Month 2-3: データ構造 │
        └───────────────────┘
                ↓
┌─────────────────────────────────────────┐
│ Month 3-4:  統計学 + アルゴリズム (並列) │
└─────────────────────────────────────────┘
                ↓
        ┌───────────────────┐
        │ Month 4-5: 機械学習 │
        └───────────────────┘
```

**特徴:**
- 🔗 スキル間の依存関係を自動で検出
- 📅 並列学習可能なスキルは同時期に配置
- ⏱️ 推定学習期間を表示
- 🎯 最短経路で目標到達

### 3. 従業員向けキャリアダッシュボード

#### 2つのアプローチ

**アプローチ1: ロールモデルから選ぶ**
```
「憧れの先輩のようになりたい」
→ 特定のメンバーを目標に設定
→ そのメンバーが持つスキルと自分のギャップを分析
→ 最適な学習経路を提案
```

**アプローチ2: 役職・職種から選ぶ**
```
「3年後に課長になりたい」
→ 課長職全体のスキル保有状況を分析
→ 50%以上の課長が持つスキル = 必須スキル
→ 30%以上の課長が持つスキル = 推奨スキル
→ 優先順位付けて学習計画を作成
```

#### 提供する機能

1. **ギャップ分析**
   - 現在のスキルと目標とのギャップを可視化
   - カテゴリ別に不足スキルを整理

2. **推薦スキルリスト**
   - 3軸スコアで評価された推薦スキル
   - 「なぜこのスキルを推薦するのか」の詳細説明
   - 前提スキルと次のステップを表示

3. **学習ロードマップ**
   - ガントチャート形式の学習計画
   - 依存関係を考慮した最適な学習順序

4. **因果グラフ可視化**
   - スキル間の因果関係を視覚的に表示
   - インタラクティブに操作可能

### 4. 組織スキルマップ

#### マネージャー向けの機能

**組織全体のスキル分布を可視化**
```
- どのスキルが組織に不足しているか
- 特定スキルの保有者は誰か
- スキルカテゴリ別の分布
- 部門別・役職別のスキル状況
```

**後継者計画（Succession Planning）**
```
- 重要ポジションの後継候補者を特定
- スキル適合度でランキング
- 育成が必要なスキルを明確化
```

**スキルギャップ分析**
```
- 組織として強化すべきスキルエリア
- 重要スキルの欠員状況
- スキルカバレッジの可視化
```

## WebUIで提供している機能

CareerNavigatorは、**Streamlitアプリ**と**React WebUI**の2つのインターフェースを提供しています。

### 🌐 Streamlitアプリ（推奨）

ブラウザベースの対話型アプリケーション。初めて使う方にはこちらがおすすめです。

#### ページ構成

**1. データ読み込み（メインページ）**
```
機能:
- CSVファイルのドラッグ&ドロップアップロード
- データ変換と品質チェック
- データの自動クレンジング
```

**2. 因果推論ベース推薦**
```
機能:
- メンバーを選択してスキル推薦
- 3軸スコアの詳細表示
- インタラクティブな因果グラフ表示
- 推薦理由の詳細説明
```

**3. 従業員向けキャリアダッシュボード**
```
機能:
- ロールモデル/役職ベースの目標設定
- ギャップ分析
- スマートロードマップ生成
- キャリアパス比較（2つの目標を並列比較）
```

**4. 組織スキルマップ**
```
機能:
- 組織全体のスキル分布可視化
- スキルギャップ分析
- 後継者計画支援
- スキル保有者マトリクス
```

### 🎨 React WebUI（モダンUI）

React + FastAPIによるモダンなWebアプリケーション。

**技術スタック:**
- フロントエンド: React 18 + TypeScript + Tailwind CSS
- バックエンド: FastAPI + Pydantic
- データ可視化: Recharts
- アイコン: Lucide React

**提供機能:**
- 従業員キャリアダッシュボード
- リアルタイムデータ可視化
- RESTful API

## 実際の使い方

### 📖 シナリオ1: 個人でキャリアパスを考える

**状況:** あなたは3年後にシニアエンジニアになりたい

**ステップ:**

1. **Streamlitアプリを起動**
   ```bash
   cd CareerNavigator
   uv run streamlit run streamlit_app.py
   ```

2. **データをアップロード**
   - メインページでCSVファイルをアップロード
   - 「データ変換を実行」ボタンをクリック

3. **キャリアダッシュボードに移動**
   - サイドバーから「2_Employee_Career_Dashboard」を選択

4. **自分を選択**
   - ドロップダウンメニューから自分の名前を選択

5. **目標を設定**
   - **方法A: ロールモデルを選ぶ**
     - 憧れの先輩社員を選択

   - **方法B: 役職を選ぶ**
     - 「シニアエンジニア」を選択
     - 最小スキル保有率: 30%（シニアエンジニアの30%以上が持つスキルを目標）

6. **ギャップ分析を確認**
   - 現在のスキルと目標とのギャップが表示されます
   - 不足しているスキルがカテゴリ別に整理されます

7. **推薦スキルを確認**
   - 3軸スコアで評価された推薦スキルリストを確認
   - 各スキルの詳細（準備完了度、確率、有用性）を確認
   - 推薦理由を読む

8. **ロードマップを確認**
   - ガントチャート形式の学習計画を確認
   - どのスキルから始めるべきか、どれを並列で学べるかが一目でわかります

9. **因果グラフを探索**
   - 推薦スキル間の因果関係を視覚的に確認
   - 「このスキルを学ぶと、次にどんなスキルが習得しやすくなるか」を理解

**結果:**
```
✅ 目標達成に必要なスキルが明確に
✅ 最適な学習順序がわかる
✅ 推定学習期間がわかる
✅ 「なぜこのスキルを学ぶべきか」が納得できる
```

### 📊 シナリオ2: マネージャーが組織スキルを分析

**状況:** あなたはチームリーダーで、組織のスキル状況を把握したい

**ステップ:**

1. **Streamlitアプリを起動**
   ```bash
   uv run streamlit run streamlit_app.py
   ```

2. **データをアップロード**
   - チーム全体のスキルデータをアップロード

3. **組織スキルマップに移動**
   - サイドバーから「3_Organizational_Skill_Map」を選択

4. **スキル分布を確認**
   - 組織全体のスキル保有状況を確認
   - カテゴリ別・タイプ別に分析

5. **スキルギャップを特定**
   - 組織として不足しているスキルを特定
   - 重要度の高いスキルの欠員状況を確認

6. **後継者計画**
   - キーポジションの後継候補者を分析
   - スキル適合度でランキング
   - 育成が必要なスキルを明確化

**結果:**
```
✅ 組織のスキル強み・弱みが明確に
✅ 採用・育成の優先順位が決まる
✅ 後継者育成計画が立てやすくなる
✅ スキルギャップが可視化される
```

### 🔄 シナリオ3: キャリアパスを比較検討

**状況:** 「データサイエンティスト」と「MLエンジニア」のどちらを目指すか迷っている

**ステップ:**

1. **キャリアダッシュボードで「比較モード」を選択**

2. **2つの目標を設定**
   - 目標A: データサイエンティストのロールモデル
   - 目標B: MLエンジニアのロールモデル

3. **並列で比較**
   - 2つのタブで切り替えながら比較
   - それぞれのギャップ分析、推薦スキル、ロードマップを確認

4. **決断材料を得る**
   - どちらが現在のスキルから近いか
   - どちらが学習期間が短いか
   - どちらが自分の興味に合うか

**結果:**
```
✅ 2つのキャリアパスを客観的に比較できる
✅ それぞれの難易度や学習期間がわかる
✅ データに基づいた意思決定ができる
```

## よくある質問

### Q1: どのようなデータが必要ですか?

**A:** 以下のCSVファイルが必要です：

```
必須データ:
- メンバーデータ (data/members/)
  - 社員コード、氏名、役職、等級など

- 習得力量データ (data/acquired/)
  - 誰がどのスキルを習得しているか

- スキルマスター (data/skills/)
  - スキルの一覧と詳細情報

オプションデータ:
- 教育データ (data/education/)
- 資格データ (data/license/)
- カテゴリデータ (data/categories/)
```

### Q2: 機密情報は大丈夫ですか?

**A:** はい、安全に管理されています。

```
✅ CSVデータはgitリポジトリに含まれません
✅ .gitignoreで自動的に除外されます
✅ データは各自のローカル環境のみに保存
✅ ディレクトリ構造のみがバージョン管理されます
```

### Q3: 因果推論は難しそうですが、詳しくなくても使えますか?

**A:** はい、専門知識がなくても使えます。

```
使う側の視点:
- 「なぜこのスキルを推薦するのか」が自然言語で説明されます
- 数式や理論を理解する必要はありません
- 直感的なUI設計で、誰でも使えます

詳しく知りたい場合:
- docs/CAUSAL_RECOMMENDATION.md で技術詳細を参照
```

### Q4: 推薦結果は信頼できますか?

**A:** 以下の理由から信頼性が高いです：

```
✅ 科学的根拠:
   - 因果推論理論に基づく（LiNGAM）
   - 学術論文で実証された手法

✅ 実データベース:
   - 組織内の実際のスキル習得パターンを学習
   - 社内のキャリアパスを反映

✅ 説明可能:
   - 推薦理由が明確
   - 3軸スコアの内訳を表示
   - 因果グラフで関係性を可視化
```

### Q5: Streamlitアプリと React WebUI、どちらを使うべきですか?

**A:** 用途に応じて選択してください：

| 用途 | 推奨 | 理由 |
|------|------|------|
| **初めて使う** | Streamlit | セットアップが簡単、機能が充実 |
| **データ分析・探索** | Streamlit | インタラクティブな分析に最適 |
| **組織スキルマップ** | Streamlit | 専用ページあり |
| **モダンなUI体験** | React WebUI | レスポンシブ、高速 |
| **API経由で利用** | React WebUI | RESTful API提供 |

### Q6: データが多い場合、動作は遅くなりますか?

**A:** 最適化されていますが、データサイズに応じた注意点があります：

```
推奨環境:
- メンバー数: ~500名
- スキル数: ~500種類
- 習得記録数: ~50,000件

パフォーマンス向上のヒント:
✅ 不要なデータは除外
✅ 分析期間を限定
✅ セッションステートのキャッシュを活用
✅ 閾値を調整して計算量を削減
```

### Q7: 学習期間の推定は正確ですか?

**A:** 目安として参考にしてください：

```
デフォルト設定:
- 1スキルあたり平均30日（1ヶ月）
- カスタマイズ可能

注意点:
⚠️ 個人の学習速度により異なります
⚠️ スキルの難易度は考慮されていません
⚠️ あくまで相対的な学習順序の把握が主目的

改善予定:
- スキル難易度の重み付け
- 個人の学習履歴からの推定
```

### Q8: エラーが出た場合はどうすれば?

**A:** トラブルシューティングガイドを参照してください：

```
よくあるエラー:

1. 「データが読み込まれていません」
   → メインページでデータアップロード

2. 「推薦結果が空」
   → 閾値を下げる（min_total_score を 0.01 など）

3. 「因果グラフが表示されない」
   → グラフ表示閾値を下げる
   → 別のスキルを選択

詳細: docs/STREAMLIT_APPS.md の
      「トラブルシューティング」セクション
```

## 🎯 まとめ

### このシステムでできること

```
個人向け:
✅ 自分に最適なスキル学習経路がわかる
✅ キャリア目標までのロードマップが見える
✅ 「なぜこのスキルを学ぶべきか」が納得できる
✅ 複数のキャリアパスを比較検討できる

組織向け:
✅ 組織のスキル分布を可視化
✅ スキルギャップを特定
✅ 後継者計画を支援
✅ 採用・育成の優先順位を決定

技術的価値:
✅ 因果推論による科学的な推薦
✅ 説明可能なAI（Explainable AI）
✅ 実データに基づく学習
```

### 次のステップ

1. **まずは試してみる**
   - Streamlitアプリを起動
   - 自分のデータで推薦を体験

2. **チームで共有**
   - このドキュメントをチーム内で共有
   - フィードバックを収集

3. **活用方法を検討**
   - 1on1での活用
   - スキル育成計画への組み込み
   - 組織開発への展開
